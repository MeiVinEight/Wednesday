# API接口

所有请求的参数与响应数据都使用JSON传递， 标准响应数据格式如下：

| 字段名     | 数据类型   | 必要 | 说明                            |
|---------|--------|----|-------------------------------|
| code    | number | Y  | 响应码，具体含义见下文                   |
| message | string | Y  | 响应消息，响应码为0时总为"OK"，响应码非0时描述错误  |
| data    | object | N  | 响应数据，由API填写，下文的API响应数据描述该字段内容 |

### 响应码表

不在下表中的响应码为未收录的响应码，具体含义参考响应消息。

| 响应码 | 说明                       |
|-----|--------------------------|
| 0   | 请求处理成功                   |
| 1   | 调用的api不存在                |
| 2   | 服务器处理api时出现异常，响应消息为异常消息  |
| 3   | 缺少api需要的参数               |
| 4   | 登录密钥无效                   |
| 5   | 未登录，具体为请求标头中未包含有效的Cookie |


### Cookie

除登录请求外的所有请求都需要验证Cookie，内容如下：

| 字段名        | 说明                   |
|------------|----------------------|
| JSESSIONID | 后端鉴权所使用的session，登录获取 |

---

## `GET /api/v1/login` 登录

这是一个特殊的API，用于登录。

首次启动时会在控制台输出登录密钥。

之后的所有请求都需要登录后才可使用，具体为，后端会验证Cookie中的JSESSIONID字段。

### 请求参数

| 字段名   | 数据类型   | 说明                                             |
|-------|--------|------------------------------------------------|
| token | string | 登录密钥的MD5字符串，如 B59C67BF196A4758191E42F76670CEBA |

### 响应数据

| 字段名     | 数据类型   | 说明                                                        |
|---------|--------|-----------------------------------------------------------|
| session | string | 登录成功时存在，表示后续请求需要用到的session，前端需要将该值设置在Cookie的JSESSIONID字段中 |

---
