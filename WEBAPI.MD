# API接口

所有请求的参数与响应数据都使用JSON传递， 标准响应数据格式如下：

| 字段名     | 数据类型   | 必要 | 说明                            |
|---------|--------|----|-------------------------------|
| code    | number | Y  | 响应码，具体含义见下文                   |
| message | string | Y  | 响应消息，响应码为0时总为"OK"，响应码非0时描述错误  |
| data    | object | N  | 响应数据，由API填写，下文的API响应数据描述该字段内容 |

### 响应码表

不在下表中的响应码为未收录的响应码，具体含义参考响应消息。

| 响应码 | 说明                       |
|-----|--------------------------|
| 0   | 请求处理成功                   |
| 1   | 调用的api不存在                |
| 2   | 服务器处理api时出现异常，响应消息为异常消息  |
| 3   | 缺少api需要的参数               |
| 4   | 登录密钥无效                   |
| 5   | 未登录，具体为请求标头中未包含有效的Cookie |
| 6   | 参数无效                     |

### Cookie

除登录请求外的所有请求都需要验证Cookie，内容如下：

| 字段名        | 说明                   |
|------------|----------------------|
| JSESSIONID | 后端鉴权所使用的session，登录获取 |

## `GET /api/v1/conn` 获取连接

获取所有连接

### 请求参数

无

### 响应数据

| 字段名     | 数据类型   | 说明     |
|---------|--------|--------|
| network | object | 网络配置结构 |

`network` 结构：

| 字段名              | 数据类型   | 说明           |
|------------------|--------|--------------|
| websocketClients | array  | WS客户端结构数组    |

`websocketClients` 的元素结构：

| 字段名    | 数据类型    | 说明                                  |
|--------|---------|-------------------------------------|
| name   | string  | 连接名，在所有连接中唯一                        |
| url    | string  | WS连接地址                              |
| token  | string  | 连接用到的令牌，不存在或为null表示不需要令牌            |
| enable | boolean | 连接是否启用，true表示已经连接到服务器，false则表示连接未建立 |

## `GET /api/v1/login` 登录

这是一个特殊的API，用于登录。

首次启动时会在控制台输出登录密钥。

之后的所有请求都需要登录后才可使用，具体为，后端会验证Cookie中的JSESSIONID字段。

### 请求参数

| 字段名   | 数据类型   | 说明                                             |
|-------|--------|------------------------------------------------|
| token | string | 登录密钥的MD5字符串，如 B59C67BF196A4758191E42F76670CEBA |

### 响应数据

| 字段名     | 数据类型   | 说明                                                        |
|---------|--------|-----------------------------------------------------------|
| session | string | 登录成功时存在，表示后续请求需要用到的session，前端需要将该值设置在Cookie的JSESSIONID字段中 |

---

## `POST /api/v1/persistence` 持久化数据API

这是一个支持前端在后端存取持久化Json数据的API，每条数据使用唯一的名字对应。

### 请求参数

| 字段名    | 数据类型    | 说明                                                        |
|--------|---------|-----------------------------------------------------------|
| action | string  | 执行的操作，可以为GET或POST，GET表示读取操作，POST表示修改操作                    |
| name   | string  | 数据的名字                                                     |
| data   | object  | 前端的持久化数据，action为POST时有效，当数据不存在时创建，否则修改；若为null或不存在该元素时表示删除 |
| auth   | boolean | 仅在action为POST时有效，设定数据是否要求登录，若为false则未登录状态下也可读取该数据         |

### 响应数据

| 字段名  | 数据类型   | 说明                         |
|------|--------|----------------------------|
| data | object | action为GET时存在，表示前端读取的持久化数据 |
